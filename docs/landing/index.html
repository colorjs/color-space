API

One object, every pair wired: space[from][to](…values) takes plain numbers in the space's conventional ranges and returns a plain array. No classes, no parsing, no state.

import space from 'color-space'

space.rgb.oklch(255, 128, 0)      // → [0.732, 0.186, 53]
space.oklch.rgb(0.65, 0.25, 180)  // any of 151 × 150 pairs

space.oklch.name    // 'oklch'
space.oklch.range   // [[0,1], [0,0.4], [0,360]]

// one space only — tree-shaken, ~2 kB
import oklch from 'color-space/oklch.js'
oklch.rgb(0.65, 0.25, 180)
// per-space metadata, generated from the sources
import meta from 'color-space/meta.js'

meta.oklch.channels    // [{symbol:'L', min:0, max:1, name:'Lightness'}, …]
meta.oklch.refs        // defining spec / paper links
meta.oklch.illuminant  // 'D65' · .observer .referred .dynamic

// whole buffers in WASM — same formulas, pinned bit-for-bit
import { alloc, convert } from 'color-space/wasm'

const buf = alloc(n)         // Float64Array(n*3) in WASM memory
convert('rgb', 'oklch', n)   // in place, zero-copy — 27 spaces

Recipes

// CSS out — values are already CSS-native
const [l, c, h] = space.rgb.oklch(r, g, b)
el.style.color = `oklch(${l} ${c} ${h})`
// gamut check — unclamped linear-RGB stays in [0, 1]
const lin = space.oklch.lrgb(0.65, 0.25, 180)
const inSRGB = lin.every(v => v >= 0 && v <= 1)
// perceptual mix — interpolate in OKLab, not in sRGB
const mix = (a, b, t) => {
  const A = space.rgb.oklab(...a), B = space.rgb.oklab(...b)
  return space.oklab.rgb(...A.map((v, i) => v + (B[i] - v) * t))
}
// camera log → display: S-Log3 footage on a Rec.709 monitor
space.slog3.rec709(0.41, 0.41, 0.41)

A conversion kernel, not a toolkit

color-space converts colors between 151 spaces — from CSS staples (OKLCH, Lab, P3) through broadcast and camera logs (YCbCr, S-Log3, LogC4, ACES) to appearance models and physical systems (CAM16, Munsell, Coloroid). Values use each space's own conventional ranges, so what you read here is what you write in CSS or in the paper.

Five guarantees hold for every space, enforced by the test suite: the canonical formula from its defining spec or paper (linked in meta.refs); a numeric pin — a cited reference value, or a differential test against colorjs.io (the CSS Color spec editors' implementation); conventional ranges on space.range; bidirectional conversion to every other space; and documented omissions — whatever is absent is listed with its reason, never silently missing. No parsing, interpolation, ΔE or gamut-mapping — that's the application layer (pair with culori or chroma-js). That restraint is why one imported space ships ~2 kB.

// npm i color-space
import space from 'color-space'

space.rgb.hsl(255, 128, 0)          // → [30, 100, 50] — CSS hsl(30 100% 50%)
space.oklch.rgb(0.65, 0.25, 180)    // ≡ CSS oklch(0.65 0.25 180)
space.slog3.rec2020(0.5, 0.5, 0.5)  // camera log → broadcast

// or one space, tree-shaken (~2 kB)
import oklch from 'color-space/oklch.js'

Not here, on purpose

Some famous systems can't be shipped honestly, so they're documented instead of faked. Licensed swatch catalogs — Pantone®, NCS®, RAL Classic, HKS®, Toyo, British Standard — are sample-defined and trademarked, with no openly redistributable CIE data; a formula can't cure that (all marks belong to their owners — visit the official sites for authoritative values). Atlas-only orders — Ostwald, DIN 6164 — have no open closed-form. Non-invertible appearance models — Hunt, Nayatani 95, LLAB, ATD95 — don't work as kernel transforms (RLAB, the one invertible early CAM, ships). Spaces marked 🕰️ in the catalog are shipped, as working history — CIE RGB 1931, NTSC 1953, S-Log 1, REDLog, Panalog, ViperLog, RYB, ANLAB…
